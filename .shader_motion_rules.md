# Motion Discipline — Shader Authoring Rules

Applies to **every** shader in this catalog and is enforced by the cron
auto-improve loop. The goal: shaders read as **soothing and smooth**, not
strobe-paced or "out of control." Default state on first selection should
feel like an installation piece, not a TikTok edit.

## 1. Speed-parameter range

Every speed/rate/spin/orbit/drift parameter must default to a **calm
floor** and reach its visible-but-tasteful peak at **MAX**, not at
DEFAULT.

| Param kind                | MIN  | MAX  | DEFAULT (calm) |
|---------------------------|-----:|-----:|---------------:|
| Camera orbit / pan        | 0.0  | 1.0  | 0.05–0.10      |
| Object drift / float      | 0.0  | 1.5  | 0.15–0.30      |
| Per-object rotation rate  | 0.0  | 1.5  | 0.10–0.30      |
| Particle / system speed   | 0.0  | 1.5  | 0.20–0.40      |
| Animation pulse rate      | 0.0  | 4.0  | 0.5–1.5        |

**Hard rule:** if a shader needs `MAX > 1.5` for any motion param,
it must call out **why** in the description (e.g. "Strobe Mode is the
point of this shader") AND default to a value at the **low end** of
the range.

## 2. Audio-react amplification ceiling

Multiplying a base motion by an audio band (`bass`, `mid`, `high`)
should **add a moderate boost**, not unleash a 5–10× speedup.

- Total motion at peak audio = `baseRate * (1 + audioBand * K)` where
  `K ≤ 1.5`. Was seeing K = 3.5 (4.5× spin) — too fast. Cap at 1.5.
- "Pulse" terms that scale a positional offset (not a speed) may use
  `K ≤ 0.6`.

## 3. Easing & transitions

Hard step changes (camera cuts, mood transitions, opacity flips) must
have an **eased ramp** of at least **15–25% of the cycle period**.

- Don't use `smoothstep(0.0, 0.05, phase)` — that's a 5%-of-period
  snap, reads as strobe.
- Use `smoothstep(0.0, 0.25, phase)` then optionally re-smooth with
  Hermite `u = u*u*(3-2*u)` for a soft ease-in/out.
- For periodic camera moves, use `cos(t * w)` inputs (already C¹) over
  step functions where possible.

## 4. Cycle period for "epoch" effects

Effects that snap to a new state (camera cuts, color cycles, lissajous
re-seeds) need a **minimum 5-second period** at audio = 0.

- `epoch = floor(t * rate)` — use `rate ≤ 0.2` for default. Was seeing
  `rate = 0.45` (2.2s period) which felt strobed.
- Audio can shorten the period — but no shorter than **3 seconds**.

## 5. Auto-tune escape hatch

If a shader genuinely needs aggressive motion (Op-art Riley, glitch
effects, intentional strobe), the description **must** open with the
phrase `"Aggressive motion: "` so reviewers/auto-improve agents skip
the calm-default rule but still cap audio amplification per §2.

## 6. Auto-improve loop enforcement

When the cron loop touches a shader it must:
1. Inventory all motion params and their defaults.
2. Flag any default outside the bounds in §1 → propose a calm value.
3. Verify audio-react multipliers vs §2 cap.
4. Replace abrupt step transitions with eased ramps per §3.
5. Slow epoch-based snap effects per §4.
6. Note any aggressive-motion shader exceptions per §5.
